Patterns give you more control over the names that get generated by defining the structure of the names and specifying certain letters.

Letter types

There are 4 kinds of supported symbol

  1. C: consonent
  2. V: vowel
  3. X: any letter
  4. lower-case letters will match that letter exactly

By default, C, V and X will all generate a cluster of one or more of their type, e.g. C could generate t, th, thr.

Modifiers

If you want to further control the pattern, you can modify the clusters using:

  1. * means optional cluster, e.g. V*CVC means the word can start with vowels, but doesn't have to
  2. {1} means exactly 1 of this letter type, e.g. CV{1} could generate Jo, but not Joe
  3. {2,4} means 2 to 4 of the letter type

Examples

  • X: any amount of any letters, no restrictions
  • aX: anything that starts with a
  • zXia: anything that starts with z ends with ia
  • CVC: a one-syllable name, like Chad, Mark or Paul
  • V*CVCVCV*C*: a long name, like Anastasia or Johnathan

Notes

  • Strict patterns make it harder to generate names. If you keep getting no results, you may need to change the pattern
  • The names generated are still based on the data source so you won't get crazy long clusters like stwvs.
  • patterns that work well for one source may not work for others, based on the structures and letters that are present in that language.
  • Y is both a vowel and a consonent
  • patterns are case-sensitive. If you want to generate names starting with v rather than a vowels, simply use lower-case v.
